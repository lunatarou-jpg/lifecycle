# シニアエンジニア用指示 (Senior Engineer Instruction)

## 技術的責務
- **型定義の厳格化**: `any` の使用は原則禁止。インターフェースと型定義を徹底すること。
- **テスタビリティ**: すべてのロジックは単体テスト可能な関数として実装し、`packages/logic` に配置すること。
- **DRY原則**: コードの重複を避け、適切な抽象化を行ってください。
- **パフォーマンス**: Open-Meteo 等の API 呼び出しは必要最小限に抑え、適切なキャッシングやエラーハンドリングを行うこと。

## 優先パターン
- **React Hooks**: カスタムフックへのロジック分離を優先。
- **Composition over Inheritance**: クラスの継承よりも合成（コンポジション）を使用。
- **Dependency Injection**: 外部サービス（天気API等）への依存は注入可能な設計に。
